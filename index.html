Om meg

Erfaren IT-arkitekt med fokus på Azure, sikkerhet, plattformdesign og automatisering. Jeg bygger løsninger med tydelig struktur, praktisk gjennomføring og et sterkt blikk for drift, kontroll og videre vekst.

Azure Landing Zones

Jeg designer Landing Zones som gir en trygg og skalerbar grunnmur for plattform, applikasjoner og videre utvikling.

  • Struktur for management groups, subscriptions og policy
  • Separasjon mellom plattform og applikasjoner
  • Nettverk, identitet og logging som en del av grunnmuren
  • Bygget for onboarding, standardisering og videre automatisering

Terraform og IaC

Infrastruktur som kode gir kontroll, repeterbarhet og mindre håndarbeid. Jeg bruker Terraform for å bygge plattform med sporbarhet og kvalitet.

  • Repeterbare deployer
  • Standardisering på tvers av miljøer
  • Bedre kontroll på endringer
  • Grunnlag for effektiv CI/CD

Azure DevOps

Jeg bygger pipelines og arbeidsflyt som gjør det enklere å deploye, dokumentere og forvalte løsninger med mindre friksjon.

  • CI/CD for plattform og applikasjoner
  • Automatisert utrulling av infrastruktur
  • Bedre kvalitet, sporbarhet og tempo
  • Støtte for sikker endringsstyring

Governance

Governance handler om å gjøre det lett å gjøre rett. Jeg bruker policy, rollefordeling og tydelige prinsipper for å bygge kontroll uten å kvele fart.

  • Azure Policy og guardrails
  • Rolle- og ansvarsdeling
  • Kostnadskontroll og standardisering
  • Driftsmodell som tåler vekst

Sikkerhet

Jeg bygger sikkerhet inn fra starten, med identitet, nettverk, policy, logging og Zero Trust som en naturlig del av plattformen.

  • Microsoft Entra ID og Conditional Access
  • Private Endpoints, segmentering og sikre kontrollpunkter
  • Defender, Log Analytics og overvåkning
  • Sikkerhet som standard, ikke som tillegg

Hybrid drift

Jeg har erfaring med både sky og tradisjonell infrastruktur. Det gjør at jeg ser helheten når løsningene må fungere på tvers av miljøer.

  • Windows Server og hybrid administrasjon
  • Identitet og tilgang på tvers av on-prem og sky
  • Drift, sikkerhet og modernisering i samme løp
  • Løsninger som kan modnes over tid

WAF / CAF

Cloud Adoption Framework og Well-Architected Framework gir et solid rammeverk for å bygge løsninger som er sikre, skalerbare og fornuftige å drifte.

CAF

  • Struktur for adopsjon, organisering og styring
  • God grunnmur for plattform og landing zones
  • Tydeligere ansvar og bedre standardisering

WAF

  • Sikkerhet, drift, kostnad, ytelse og robusthet
  • Arkitektur som kan forklares og videreutvikles
  • Prinsipper som tåler vekst og endring
